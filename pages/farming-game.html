Farming Game

A C# farming simulation built to demonstrate object-oriented design, persistent JSON save/load, and a simple in-game economy through a playful and structured gameplay loop.

View Source

Overview

Echoes of Lemnos was our final project for our Object-Oriented Programming class, and we were given very little guidance in terms of structure or requirements. Because of that, our team decided to create a game in Unity, which became an exciting challenge and a meaningful way to apply the concepts we had learned. It was also a major transition from Visual Studio Code into a new environment, but we quickly discovered that core OOP principles were still highly relevant and very useful in Unity C# for 2D development.

  1. Planning the gameplay loop and core requirements
  2. Designing the class structure and game entities
  3. Implementing the save system and economy logic
  4. Testing persistence, inventory flow, and gameplay balance
  5. Refining the experience and polishing the interaction flow

Challenges & Solutions

Challenge

Designing a save system that preserved player progress and inventory across sessions.

Solution

Implemented JSON serialization to store game state and reload it cleanly between runs.

Challenge

Keeping the game logic modular and easy to manage as features expanded.

Solution

Separated responsibilities into distinct classes for entities, inventory, and save management.

What I Learned

One of the hardest parts of this project was finding quality learning material for Unity, since resources online were often limited or highly specific to particular cases. Eventually, we consulted our professor and found courses that helped us understand the tools and workflows much more clearly. That experience taught me the importance of reaching out for help when needed and being proactive about learning from outside resources.
